Settings

This prototype calls the selected provider's API directly from your browser. Your key is kept in this browser's local storage only — it is sent only to the provider you choose. A separate key is remembered per provider.
Instructor use only. Keys in the browser are acceptable for personal testing, but this pattern must never be used in a student-facing deployment — the production platform proxies every provider through a backend. Never commit a key to the repository.

Session metrics

Active time in the studio for this session — idle gaps over 3 minutes and time with the tab hidden are not counted. Metrics are saved with the session file and continue when it is reopened.
Engagement numbers are context, not assessment. The meaningful record of a student's work is the artifact history and the reasoning in the conversation — read those, not these.

Assignment brief

Paste a coursework scenario, or upload it as a file — text, PDF, or an image (e.g., a scanned brief). The tutor grounds its questioning in the brief while still making the student do the analysis. Load it before or during a session; clear it to run brief-free.
